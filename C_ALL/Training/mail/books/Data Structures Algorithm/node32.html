Constructors, Destructors and Dynamic Data

In the above example we defined constructors and destructors. These are generally required when dealing with dynamic data (ie, allocated with new). Constructors are also useful when some initialisation is required. The namedstack constructor both allocated memory and did some initialisation; the destructor just de-allocated memory. When NOT using dynamic data, constructors/destructors are not strictly required. However, it is probably good style to always include then, and just make them empty when nothing is required.

In this section we'll look a little more closely at constructors, destructors and dynamic data.
